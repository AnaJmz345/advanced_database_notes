## What I understood
- UNION combines the results of two queries into a single result set.
- Some rules it follows are:

    - Both queries must return the same number of columns
    - The columns must have compatible data types
    - Duplicates are automatically removed

- Example:
Query A → [1,2,3]
Query B → [3,4,5]

UNION

Result → [1,2,3,4,5]
- DISTINCT removes duplicate rows from a result.

UNION ALL

- UNION ALL also combines the results of two queries, but it does NOT remove duplicates.
Because it doesn't check for duplicates, it is usually faster than UNION.
- Example:
Query A → [1,2,3]
Query B → [3,4,5]

UNION ALL

Result → [1,2,3,3,4,5]

Set difference

- Set difference means finding rows that exist in one query but not in another.In SQL this is usually done with MINUS.
- Example:
Query A → [1,2,3,4]
Query B → [3,4]

Difference (A - B)

Result → [1,2]

MINUS
- MINUS returns rows that exist in the first query but not in the second query.It also removes duplicates automatically.
Query A → [1,2,3,4]
Query B → [3,4]

MINUS

Result → [1,2]

INTERSECT
- INTERSECT returns only the rows that appear in both queries.Duplicates are also removed automatically.
- Example:
Query A → [1,2,3,4]
Query B → [3,4,5]

INTERSECT

Result → [3,4]
 
## Something new I learned:

Symmetric Difference with Group By
Another way to compute symmetric difference is using UNION ALL with GROUP BY.

The idea is to combine both tables, count how many times each row appears and keep rows that appear only once
