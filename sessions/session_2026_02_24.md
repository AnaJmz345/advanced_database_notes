## What I understood
- An aggregation function applies the function to the specified column and it summarizes in 1 value because it aggregates all the vaalues in 1. Some examples are the functions max,min,avg,sum,count
- All aggregation applies after the where. You cannot put WHERE avg(price)>6 because the where is applied befor the aggregation function and, so if you try the filter the average, it will throw an error because the average hasn't be calculated yet. That's why we use the HAVING aavg(price)>6 instead of where to filter results from aggregate fucntions, these aggregate are executed before the having.

- Example in class:
SELECT
    AVG(CASE 
            WHEN category = 'home' THEN price 
        END) AS avg_home_prices,

    AVG(CASE 
            WHEN category = 'office' THEN price 
        END) AS avg_office_prices,

    AVG(CASE 
            WHEN price > 5 THEN price 
        END) AS avg_bigger_prices
FROM products;

## Something new I learned:

By default aggregate functions use every input value. But most allow you to work on the unique values in the input. You do this with the keyword distinct. You can also use distinct in most stats functions, like sum and avg. 

For example, you can find the number of different values in the colour column by passing "distinct colour" to count. There are three colours (red, green, and blue). So doing this returns three:

```
select count ( distinct colour ) number_of_different_colours
from   bricks;

```

You can explicitly tell the function to process every row with the keyword all. You can also use unique as a synonym for distinct


- GROUP BY, ROLL UB, CUBE
A regular GROUP BY returns only the exact grouped combinations of the specified columns, without any subtotals or grand totals. ROLLUP extends GROUP BY by adding hierarchical subtotals from right to left, producing totals for progressively higher levels of aggregation and ending with a grand total (it generates N+1 grouping levels). CUBE, on the other hand, generates subtotals for every possible combination of the grouped columns, including totals per individual column and the grand total (it generates 2^N grouping combinations).


Example table:

| colour | shape  |
| ------ | ------ |
| red    | cube   |
| red    | sphere |
| red    | cube   |
| blue   | cube   |

GROUP BY

select colour, shape, count(*)
from bricks
group by colour, shape;

| colour | shape  | count |
| ------ | ------ | ----- |
| red    | cube   | 2     |
| red    | sphere | 1     |
| blue   | cube   | 1     |

ROLLUP - Adds hierarchical subtotals (right to left).

select colour, shape, count(*)
from bricks
group by rollup(colour, shape);

| colour | shape  | count                     |
| ------ | ------ | ------------------------- |
| red    | cube   | 2                         |
| red    | sphere | 1                         |
| red    | NULL   | 3     ← subtotal for red  |
| blue   | cube   | 1                         |
| blue   | NULL   | 1     ← subtotal for blue |
| NULL   | NULL   | 4     ← grand total       |

CUBE - Generates every possible subtotal combination.

select colour, shape, count(*)
from bricks
group by cube(colour, shape);

| colour | shape  | count                       |
| ------ | ------ | --------------------------- |
| red    | cube   | 2                           |
| red    | sphere | 1                           |
| red    | NULL   | 3     ← subtotal per colour |
| blue   | cube   | 1                           |
| blue   | NULL   | 1                           |
| NULL   | cube   | 3     ← subtotal per shape  |
| NULL   | sphere | 1                           |
| NULL   | NULL   | 4     ← grand total         |
